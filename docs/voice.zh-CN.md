# 语音 mod 使用指南（小白向）

> 这篇文档教你：在用了 **ZstdNet** 的服务器上，怎么让 **Simple Voice Chat、Plasmo Voice** 这类「语音 mod」正常说话。
> 不需要任何网络基础，照着做就行。看不懂的名词，文末有[术语表](#术语表)。

ZstdNet 是一个**压缩 Minecraft 联机流量、省带宽**的 mod。它会接管你和服务器之间的连接——这在以前会**意外挡住语音 mod 的端口**，导致进了服却听不到 / 说不了话。
**现在 ZstdNet 已经内置「零配置兼容语音 mod」**，绝大多数情况下你**什么都不用设**，语音就能自己通。这篇文档就是告诉你那「绝大多数情况」怎么做，以及万一不通怎么排查。

---

## 30 秒速览：我到底要做什么？

| 你的身份 | 你要做的事 |
|---|---|
| **玩家**（只是想进别人的服说话） | ① 装好 **ZstdNet 客户端 mod** ② 装好**和服务器一样的语音 mod**（如 Simple Voice Chat）③ 正常进服。**语音自动就通，不用配置任何端口。** |
| **服主**（自己开服给别人玩） | 装好 ZstdNet（mod 版或插件版）+ 语音 mod，**默认就是零配置**。公网服只需要在防火墙/端口映射里**放行你的入口端口（TCP+UDP 都放行）**这一个口。详见 [服主怎么用](#服主怎么用)。 |

> **最常见的组合**：公网服 + Simple Voice Chat。直接看 [玩家怎么用](#玩家客户端怎么用) 和 [服主怎么用 → mod 服 / 插件服](#服主怎么用) 就够了。

## 先搞懂几个最基础的概念（小白必读）

如果你已经懂端口、UDP、公网 IP，可以跳过这节。

- **端口（Port）**：可以把服务器想成一栋楼，**IP 是楼的地址，端口就是楼里一个个房间号**。游戏走一个房间（Minecraft 默认 `25565`），语音 mod 通常走另一个房间（Simple Voice Chat 默认 `24454`）。
- **TCP 和 UDP**：两种「快递方式」。**游戏本体走 TCP**（稳，要签收）；**语音走 UDP**（快，丢一两个包无所谓，适合实时通话）。放行端口时这俩是分开算的，所以下文会强调「**TCP 和 UDP 都要放行**」。
- **公网 IP / 端口映射（端口转发）**：让外网的人能连到你家/你服务器，需要把某个端口「对外开放」。在云服务器叫**安全组放行**，在家用宽带叫**路由器端口映射 / 端口转发**，在内网穿透工具（FRP、樱花等）叫**穿透/映射一个端口**。三者是一回事：**把某个端口号对外打开**。
- **语音 mod 为什么以前会坏？**
  语音 mod 的工作方式是：你进服后，服务器悄悄告诉你的客户端「**语音去 xxx 端口**」，你的语音 mod 就往那个端口发声音。
  而 ZstdNet 为了压缩，会把你的连接「改道」到本机中转。结果语音 mod 把声音发到了一个**没人接听的端口**，于是听不到、说不了。
- **ZstdNet 现在怎么修好的？**（你不用懂细节，知道结论即可）
  服务器端的 ZstdNet 会**自动找出**语音 mod 在用哪个端口，玩家一进服就**自动通知**客户端的 ZstdNet，客户端**自动在本机把那个端口接上**。全程不用你手动填端口。

## 各类语音 mod 一览

| 语音 mod | 默认端口 | 开箱即用吗？ | 说明 |
|---|---|---|---|
| **Simple Voice Chat**（最常见） | `24454`（独立端口，UDP） | ✅ 现在自动兼容 | 这是 ZstdNet 这次主要解决的对象。默认用独立端口 24454，以前会被挡，现在自动接管。 |
| **Plasmo Voice** | **默认跟随游戏端口**（同端口） | ✅ 一直都能用；改了端口也自动兼容 | 默认就和游戏走同一个端口，本来就不受影响。若管理员给它单独设了端口，现在也会被自动接管。 |
| **同端口类**：Sable、机械动力·航空学（Create Aeronautics）等 | 跟游戏同一个端口 | ✅ 一直都能用 | 它们的 UDP 本来就和游戏走同一个口，由 ZstdNet 内置直通覆盖，无需任何额外处理。 |
| **其它 / 冷门 UDP mod** | 各不相同 | ⚠️ 可能需要手动补一行 | 自动探测覆盖不到时，服主在配置里用 `extra_udp_ports` 手填端口即可（见下文）。 |

> 一句话：**装了 Simple Voice Chat 的服，现在直接就能语音**；Plasmo / Sable / 航空学本来就没问题。

## 玩家（客户端）怎么用

这是最简单的部分，**只有三步，且不需要配置任何端口**：

1. **装 ZstdNet 客户端 mod**：和你平时装 mod 一样，丢进 `mods/` 文件夹。要和服务器同一个游戏版本 + 同一个加载器（Forge / NeoForge / Fabric）。
2. **装语音 mod**：装**和服务器一样的语音 mod**（比如服务器用 Simple Voice Chat，你也装 Simple Voice Chat），版本对应。
3. **正常进服**：
   - 如果是 **mod 服**：地址端口和平时一样填（和不装 ZstdNet 时一模一样）。
   - 如果是 **插件服（Paper/Spigot 等）**：服主会告诉你一个「**ZstdNet 入口端口**」（通常是游戏端口 + 1，例如 `服务器IP:25566`），把它填进服务器地址即可享受压缩。

进服后语音就**自动连通**了。你**不需要**在语音 mod 里改任何「服务器地址 / 端口」设置。

> 想确认压缩有没有生效？进游戏后输入指令 `/zstdhud on`，屏幕上会显示压缩状态。这和语音无关，只是好玩。

## 服主怎么用

核心结论：**默认配置就是零配置兼容语音，你基本不用动。** 唯一需要操心的是**防火墙 / 端口放行**。

ZstdNet 有两种部署形态，按你的服务器类型二选一：

### A) mod 服（Forge / NeoForge / Fabric）

1. 在 `mods/` 里装好 **ZstdNet** 和你的**语音 mod**（如 Simple Voice Chat）。
2. 正常启动服务器。ZstdNet 会自动接管端口（`auto_takeover`），并**自动探测**语音 mod 的端口。
3. 配置文件在 `config/zstdnet-server.properties`，**默认就是 `voice_transport=tunnel`（隧道模式）**——语音和游戏**共用同一个对外端口**。
4. **放行端口**：只需要在防火墙 / 端口映射里放行你的**对外入口端口**（通常就是 `server.properties` 里的 `server-port`，如 `25565`），**TCP 和 UDP 都要放行**。搞定。

### B) 插件服（Paper / Spigot / Purpur / Arclight / Mohist 等）

1. 把 **`zstdnet-bukkit-<版本>.jar`** 丢进 `plugins/`，把语音**插件**（如 Simple Voice Chat 的 Bukkit 版）也装好，启动。
2. 首次启动会生成 `plugins/ZstdNet/zstdnet-server.properties`，默认：
   ```properties
   auto_takeover=false
   listen=0.0.0.0:25566      # ZstdNet 对外入口端口（默认 = server-port + 1）
   target=127.0.0.1:25565    # 你本机的 Minecraft 后端端口
   voice_transport=tunnel    # 默认隧道：语音和游戏共用入口端口
   ```
3. ZstdNet 插件会自动扫描 `plugins/voicechat/`、`plugins/PlasmoVoice/` 找到语音端口，玩家进服时**自动下发**给客户端（对没装 ZstdNet 的原版玩家无任何影响）。
4. **放行端口**：放行 `listen` 端口（如 `25566`），**TCP + UDP 都放行**。原版玩家继续用 `25565`，装了 ZstdNet 的玩家用 `25566`。

> 插件服小贴士：如果你是**先装了 ZstdNet 插件、之后才装语音插件**，重启后语音端口可能还没被探测到。执行一次 `/zstdnet reload`，或重启服务器即可。

### tunnel 还是 bridge？（两种语音传输方式）

配置项 `voice_transport` 控制语音怎么走，**默认 `tunnel`，新手用默认即可**：

| 模式 | 怎么走 | 你要放行几个端口 | 适合谁 |
|---|---|---|---|
| **`tunnel`（默认，推荐）** | 语音 UDP **塞进游戏的同一个入口端口**，由服务端拆分转发 | **只放行入口端口一个**（TCP+UDP） | 绝大多数人，尤其是 **FRP / 内网穿透 / 只能开一个口**的场景 |
| **`bridge`** | 客户端把语音 UDP **直连真实服务器的语音端口** | 入口端口 + **语音端口单独再放行一个**（UDP） | 你能随便开端口，且想让语音少绕一道中转 |

打个比方：**tunnel = 语音和游戏走同一扇门**（省事）；**bridge = 给语音单独开一扇门**（要多放行一个端口）。

要切换成 bridge，就在配置文件里改成 `voice_transport=bridge`，保存后服务器会在几秒内自动热重载（或 `/zstdnet reload`）。改成 bridge 后，记得**把语音端口（如 24454/UDP）也单独对外放行**。

### 防火墙 / 端口放行 到底怎么操作？

- **云服务器（阿里云/腾讯云/AWS 等）**：在控制台的「**安全组**」里加一条规则，放行你的入口端口，协议选 **TCP 和 UDP（或 全部协议）**。
- **家用宽带**：登录路由器后台，找「**端口转发 / 虚拟服务器 / NAT**」，把入口端口转发到你这台开服的电脑，**TCP+UDP**。
- **内网穿透（FRP、樱花穿透、ngrok 等）**：映射/穿透你的入口端口即可。**用 tunnel 模式时只穿一个口就够了**——这正是 tunnel 模式最大的好处。

## 配置项速查（服主）

都在 `zstdnet-server.properties` 里（mod 服在 `config/`，插件服在 `plugins/ZstdNet/`）：

- **`voice_transport`**：语音传输方式，`tunnel`（默认）或 `bridge`。见上文。
- **`extra_udp_ports`**：手动补充需要透传的 UDP 端口，逗号分隔。**自动探测覆盖不到的冷门语音/UDP mod 用这个兜底**。例：`extra_udp_ports=24454,30000`。留空表示只靠自动探测。
- **`voice_chat_passthrough`**：是否启用语音 UDP 转发，默认 `true`。**设成 `false` 会彻底关闭语音处理**（不探测、不接管、不下发）——一般不用动。

> 语音流量**永远不压缩**，只做原样转发（声音本来就不太压得动，压了反而增加延迟）。压缩只作用于游戏本体。

## 常见问题排查

**Q：进服了，但听不到 / 说不了话。**
1. 确认**客户端和服务器装了同一个语音 mod、版本对应**。
2. 确认服主**放行了入口端口，且 TCP+UDP 都放行了**（最常见的坑就是只放行了 TCP，UDP 忘了）。
3. 插件服：先装插件后装语音插件的，执行一次 `/zstdnet reload` 或重启。
4. 看服务器后台日志有没有 `voice ports detected ...` 或 `WARN ... voice_host ...` 字样（见下「已知边界」）。

**Q：语音只能单向 / 断断续续。**
- 多半是 **UDP 没放行干净**或网络丢包。先确认 UDP 放行；用 FRP/穿透的话确认穿透的是 UDP（或 TCP+UDP）。

**Q：我用的是冷门语音 mod，自动探测不到。**
- 服主在配置里加 `extra_udp_ports=<那个 mod 的端口>`，保存后 `/zstdnet reload`。

**Q：Plasmo / Sable / 航空学 需要特殊设置吗？**
- 不需要。它们默认和游戏同端口，本来就通。

## 已知边界（什么时候会失效）

ZstdNet 接管语音的前提是：**语音 mod 让客户端把声音发到「服务器的连接地址」**（默认就是这样）。
**唯一会失效的情况**：管理员**手动**把语音 mod 的「对外公网地址」写成了一个**具体的公网 IP**——

- Simple Voice Chat：`voicechat-server.properties` 里的 `voice_host` 被填了公网地址；
- Plasmo Voice：`config.toml` 里 `[host.public]` 的 `ip` 被填了具体公网地址（非 `0.0.0.0`）。

这时客户端会**按你填的地址直连、绕过 ZstdNet**（服务端日志会打一条 `WARN` 提醒）。这是管理员主动要求的「直连」，和「经 ZstdNet 中转」互斥。
**想让 ZstdNet 接管语音，就把它们留默认（留空 / `0.0.0.0`）。**

## 术语表

- **mod 加载器**：Forge / NeoForge / Fabric，运行 mod 的底座。客户端和服务器要用同一种。
- **mod 服 / 插件服**：mod 服用 mod 加载器跑（mods 文件夹）；插件服是 Paper/Spigot 这类用插件的服（plugins 文件夹）。混合端（Arclight/Mohist）两者都吃，推荐直接用 ZstdNet 插件版。
- **端口（Port）**：服务器上的「房间号」，区分不同服务。游戏、语音各用各的端口。
- **TCP / UDP**：两种网络传输方式。游戏走 TCP，语音走 UDP。放行端口时要分别考虑。
- **入口端口 / listen 端口**：ZstdNet 对外暴露、玩家实际连接的端口。
- **后端端口 / target**：你本机真正的 Minecraft 服务器端口，只在本机内部用。
- **端口映射 / 端口转发 / 放行**：把某个端口对外打开，让外网能连进来。
- **tunnel（隧道）/ bridge（桥接）**：ZstdNet 的两种语音传输方式，见上文。
- **公网直连 / FRP / 内网穿透**：让外网玩家连到你服务器的几种方式。
