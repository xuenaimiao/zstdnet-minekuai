# ZstdNet 新手使用指南（零基础也能看懂）

> 这份文档是写给**完全没接触过 mod / 开服**的新手的。全程用大白话，一步一步教你用上 ZstdNet。
> 想看更详细的参数说明，再去看 [`README.zh-CN.md`](../README.zh-CN.md)（那是给进阶用户的完整手册）。

---

## 一、这是什么？（一句话 + 一个比喻）

**ZstdNet 帮你的 Minecraft 联机「压缩流量」，省公网带宽、减少卡顿。**

打个比方：你要往外地寄一大箱衣服，先抽真空压扁再寄，运费就省了；对方收到再拆开，衣服一件不少。
游戏数据也一样 —— 发出去之前先压一压，对面收到再还原，**你看到的画面、玩到的内容完全一样**，只是中间走的网络流量小了很多。

适合这些情况：
- 玩**整合包服**、**机械动力（Create）服**这类数据重复多的服；
- 用 **frp / 内网穿透**（在家用电脑开服、让外面的朋友进）的服；
- 公网**带宽小**、或者**按流量收费**的服。

> ⚠️ **最重要的一点**：压缩是「两头的事」。发的人要压、收的人要解，**两边都装了才有效**。只有一边装，等于没装。

## 二、先弄清楚「你是谁」——这决定你要装哪个

| 你的情况 | 你要装的东西 |
| --- | --- |
| 我是**普通玩家**，想进一个装了 ZstdNet 的服 | 装 **客户端 mod**（见[第三节](#三玩家篇怎么进一个-zstdnet-服)） |
| 我开的是 **Forge / NeoForge / Fabric 服**（需要 mod 加载器） | 服务端也装 **同款 mod**（见[第四节-A](#a-modforgeneoforgefabric)） |
| 我开的是 **Paper / Spigot / Purpur 服**（插件服，没有 mod 加载器） | 装 **插件版**（见[第四节-B](#b-插件服paperspigotpurpur)） |
| 我开的是 **Arclight / Mohist 等混合端** | 插件版或 mod 版都行，推荐插件版（见[第四节-C](#c-混合端arclightmohist-等)） |

### 名词小课堂（看不懂上面表格再看这里）

- **客户端 / 服务端**：你自己玩游戏的电脑叫「客户端」；大家一起连进去的那台叫「服务端 / 服务器」。
- **mod 加载器**：能让游戏加载 mod 的底座，常见的有 **Forge、NeoForge、Fabric**。装 mod 必须先有它。
- **插件服**：用 **Paper / Spigot / Purpur** 开的服。它们**不吃 mod**，只吃「插件（plugin）」。
- **混合端**：**既能吃 mod 又能吃插件**的服，比如 Arclight、Mohist。
- **端口**：可以理解成「服务器的门牌号」。连服务器时那串 `地址:数字` 里的数字（比如 `25565`）就是端口。

## 三、玩家篇：怎么进一个 ZstdNet 服

### 第 1 步：搞清楚你的游戏版本和加载器

打开 Minecraft 启动器，看你要玩的那个版本是 **Forge / NeoForge / Fabric** 的哪个，以及版本号（比如 `1.20.1`）。
本仓库目前提供这些客户端：**Forge 1.19.2、Forge 1.20.1、NeoForge 1.20.1、NeoForge 1.21.1、Fabric 1.20.1、Fabric 1.21.1**。

### 第 2 步：把客户端 mod 放进 `mods` 文件夹

1. 下载和你「加载器 + 版本」对应的那个 ZstdNet 文件（`.jar`）。
2. 找到游戏的 `mods` 文件夹：
   - 一般在 `.minecraft\mods`（用整合包启动器的话，在那个整合包实例的目录里有 `mods`）。
   - 找不到就先在游戏里随便装个加载器启动一次，它会自动生成 `mods` 文件夹。
3. 把下载的 `.jar` 直接拖进 `mods` 文件夹。
4. 用对应的加载器版本启动游戏。

### 第 3 步：进服

- **如果服主开的是 mod 服或混合端**：地址端口和平常一样，直接连服主给你的地址即可。
- **如果服主开的是插件服**：地址里的**端口可能不一样**（比如平常是 `25565`，要你连 `25566`）。**以服主告诉你的为准**。

### 第 4 步：确认「压缩真的生效了」

进服后，游戏里输入指令 `/zstdhud`，会出现一个小面板，显示压缩比等信息。
**压缩比越小越好**（比如 30% 表示流量压到了原来的三成）。看到面板有数据在动，就说明生效了。

> ❓ **不装 mod 能进吗？** 能 —— 走普通端口照常进服，只是**没有压缩**而已，不影响你玩。

## 四、服主篇：怎么给自己的服开启 ZstdNet

### A. mod 服（Forge / NeoForge / Fabric）

1. 把**服务端对应版本**的 ZstdNet `.jar` 放进服务器的 `mods` 文件夹。
2. 正常启动服务器。**默认就能用**：它会自动接管你 `server.properties` 里的 `server-port`，玩家**继续连原来的端口、原来的地址**，什么都不用改。
3. （可选）想调参数？第一次启动后会生成 `config/zstdnet-server.properties`，里面每一行都有中文注释。**一般保持默认即可。**

> 默认 `auto_takeover=true` 的意思是：ZstdNet 自己把对外端口接管了，再悄悄把真正的游戏服挪到本机另一个端口，**你不用手动配端口**。

### B. 插件服（Paper / Spigot / Purpur）

> 关键区别：插件是在服务器**启动之后**才加载的，那时端口已经被占住了，插件**没法**像 mod 那样「接管原端口」。
> 所以插件版用一个**单独的端口**。

1. 把 `zstdnet-bukkit-x.x.x.jar` 放进服务器的 `plugins` 文件夹（**同一个 jar 同时支持 1.20.1 和 1.21.x**，不用区分版本）。
2. 启动服务器。第一次会生成 `plugins/ZstdNet/zstdnet-server.properties`，默认长这样：
   ```properties
   enabled=true
   auto_takeover=false
   listen=0.0.0.0:25566      # ZstdNet 对外入口（默认 = 游戏端口 + 1）
   target=127.0.0.1:25565    # 你本机真正的游戏端口
   ```
3. **在防火墙 / 服务器安全组里放行 `listen` 端口**（例子里是 `25566`）。
4. 告诉玩家：
   - **装了 ZstdNet 的玩家** → 连 `你的IP:25566`（即 `listen` 端口），享受压缩；
   - **没装 mod 的原版玩家** → 继续连 `你的IP:25565`（原游戏端口），照常进服、不受影响。

**管理指令**（需要 OP 权限）：

| 指令 | 作用 |
| --- | --- |
| `/zstdnet status` | 查看运行状态、端口、连接数、压缩比 |
| `/zstdnet reload` | 改完配置后重新加载（改配置文件后几秒内也会自动热重载） |
| `/zstdnet start` | 启动代理 |
| `/zstdnet stop` | 停止代理 |

> 小提示：`listen` 端口必须和游戏端口**不一样**、而且是空闲的。需要改就直接编辑那一行。插件服请保持 `auto_takeover=false`。

### C. 混合端（Arclight / Mohist 等）

混合端底层是 Forge/NeoForge，**既能放 mod 也能放插件**，两种都行：

- **最省心**：直接用**插件版**（丢进 `plugins/`），不依赖混合端对底层注入的兼容性。
- 也可以用对应的 **mod 版**（丢进 `mods/`）。如果遇到自动接管端口失败，就把 mod 配置里的 `auto_takeover` 改成 `false`，再手动填 `listen` / `target`，或者干脆改用插件版。

## 五、进阶一点点：frp / 内网穿透怎么配（看不懂可跳过）

**frp / 内网穿透**就是把你家电脑上的某个端口「映射」到公网，让外面的人能连进来。

配的时候记住一句话：**让公网隧道最终指向 ZstdNet 的入口端口，而不是原始游戏端口**，否则就绕过了压缩。

典型链路：

```
玩家  →  公网 frp 端口  →  你家电脑上的 ZstdNet 入口端口  →  本机游戏端口
```

如果整合包里有 **Sable / 机械动力：航空学** 这类「同端口 UDP」的 mod，记得让隧道**同时转发 TCP 和 UDP**。语音 mod（如 Simple Voice Chat）ZstdNet 也已自动兼容，一般无需额外配置。

## 六、常见问题（新手最担心的）

**Q：会被封号 / 算作弊吗？**
不会。ZstdNet 只是压缩你和服务器之间传输的数据，**不改任何游戏逻辑、不动游戏内容**，对服务器来说收到的东西和原版一模一样。

**Q：会让我更卡吗？**
压缩本身吃的 CPU 很小，主要作用是**省带宽**。默认压缩等级 `level=9`，已是兼顾效果和开销的设置。

**Q：没装 mod 的朋友还能进我的服吗？**
能。他们走原始游戏端口照常进，只是没有压缩。

**Q：`/zstdhud` 显示压缩比接近 100%，是没生效吗？**
压缩比 100% 表示「这段数据本来就压不动」（比如已经是压缩过的内容）。多玩一会儿、数据多了，比例通常会降下来；只要面板有数据，就说明在工作。

**Q：进不去服怎么办？**
按这个顺序排查：① 地址和端口对不对（插件服要用服主给的 `listen` 端口）；② 你的加载器和版本和服务端是否匹配；③ 服主那边 `listen` 端口有没有在防火墙放行。

**Q：和别的联机 mod / 语音 mod 冲突吗？**
ZstdNet 工作在网络传输层，正常情况下不与玩法 mod 冲突；Simple Voice Chat 等基于 UDP 的语音 mod 已自动兼容。

## 七、超简名词表

| 名词 | 大白话解释 |
| --- | --- |
| 客户端 | 你自己玩游戏的电脑 |
| 服务端 | 大家一起连进去的那台服务器 |

| HUD | 游戏里那个显示信息的小面板（`/zstdhud` 开关） |
