# 玩家怎么用

跟着做，大约 5 分钟。目标：**装上客户端 → 连服 → 确认在省流量**。

> 前提：你想连的服务器，服主已经装好了 ZstdNet（服务端 / 插件）。如果服主没装，你装了也压不了。

---

## 第一步：搞清楚你的「游戏版本 + 加载器」

ZstdNet 的 jar 文件分版本，**必须和你游戏对得上**，否则进不去游戏。

怎么看？打开 Minecraft 启动器（官启 / PCL2 / HMCL 等），看你要玩的那个「版本」名字，通常长这样：

- `1.20.1-forge-47.x.x` → 版本 **1.20.1**，加载器 **Forge**
- `fabric-loader-…-1.21.1` → 版本 **1.21.1**，加载器 **Fabric**
- `neoforge-…-1.21.1` → 版本 **1.21.1**，加载器 **NeoForge**

记下这两样：**版本号** + **加载器**。

> 还没装加载器？先去装 Forge / Fabric / NeoForge 其中一个（按你整合包要求的来）。这一步不是 ZstdNet 的内容，
> 网上「Minecraft 怎么装 Forge / Fabric」教程很多，装好后启动器里会多出对应的版本可选。

---

## 第二步：下载对应的客户端 jar

从你拿到 ZstdNet 的地方（发布页 / 群文件 / 服主提供），下载**和你第一步对应**的那个文件，例如：

- 你是 Fabric 1.21.1 → 下 `zstdnet-fabric-1.21.1-*.jar`
- 你是 Forge 1.20.1 → 下 `zstdnet-forge-1.20.1-*.jar`

> ⚠️ 别下错：Forge 的 jar 不能放进 Fabric，版本也不能混。下错了游戏会启动失败或闪退。

---

## 第三步：放进 `mods` 文件夹

1. 打开游戏的 `mods` 文件夹（启动器里一般有「打开版本文件夹 / mods 文件夹」按钮；
   或手动找 `.minecraft/versions/<你的版本>/mods/`，整合包通常是版本隔离的）。
2. 把刚下载的 jar **直接拖进去**。
3. 不用解压，不用改名。

---

## 第四步：启动游戏，连接服务器

ZstdNet 会**自动工作**，你基本不用做额外设置。

- **大多数情况**：服主用的是默认推荐设置，你**还是连原来那个地址**就行，比如：
  ```text
  play.example.com:25565
  ```
  ZstdNet 会在你电脑本地悄悄开个临时代理、接管这条连接并压缩，你无感。

- **少数情况**：如果服主告诉你「要连某个特定端口」（比如插件服常见的 `:25566`），那就按服主给的地址连。

进服正常游玩即可。

---

## 第五步：确认真的在压缩（可选但推荐）

进游戏后，在聊天框输入指令：

```text
/zstdhud
```

会调出一个 **HUD 面板**，上面有 **Ratio**（压缩比）：

- **Ratio 越低越好**（比如 8%、20%），说明压缩生效、很省流量。
- 如果 Ratio 接近 **100%**，多半是这台服务器没装 ZstdNet 服务端，或这条连接没走压缩 → 见 [常见问题](faq.md)。

HUD 开关用法：

```text
/zstdhud          切换显示/隐藏
/zstdhud on       打开
/zstdhud off      关闭
```

---

## 想更省流量？

如果服主开了进阶功能（字典、实体变换 transform 等），你客户端这边**也可能要相应打开一个开关**才能吃到收益。
详见 👉 [省流量进阶设置](save-bandwidth.md)。客户端的设置文件在：

```text
config/zstdnet-client.toml
```

---

## 语音模组会受影响吗？

不会。ZstdNet 兼容 Simple Voice Chat、Plasmo 等常见语音模组，以及 Sable / 机械动力：航空学这类
「同端口 UDP」模组——**零配置自动兼容**，你正常用就行。如果语音异常，见 [常见问题](faq.md)。

---

## 常见小问题

- **进不去游戏（启动就崩）**：八成是 jar 和你的版本/加载器不匹配，回到第一步核对。
- **进游戏但进不去服务器**：见 [常见问题与排错](faq.md)。
- **找不到 `/zstdhud` 指令**：确认 jar 真的装进了**你正在玩的那个版本**的 mods 文件夹。
